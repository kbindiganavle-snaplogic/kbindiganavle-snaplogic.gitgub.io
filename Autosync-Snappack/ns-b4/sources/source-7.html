644      *

646      *    the middle of the SQL string input.
647      *
648      * The expression looks for FROM clause in the syntax and is not preceded

661      * the match.

665      *

1130     /*
1131          Explanation for the LIMIT_CLAUSE_PATTERN:
1132 
1133          Looking at the regex it might look slightly complicated, but let us break it down:
1134 
1135          There are two parts:
1136          1. Just with the LIMIT clause i.e. (?i) +limit +[0-9]+ *$
1137          2. With the LIMIT and OFFSET clause i.e. (?i) +limit +[0-9]+ +offset +[0-9]+$
1138 
1139          1. (?i) +limit +[0-9]+ *$ explanation
1140             a. (?i) basically means case insensitive. LIMIT keyword can be in any case.
1141 
1142             b. <space> followed by plus indicates that there can be at least one or more
1143             blank spaces before the LIMIT keyword starts.
1144 
1145             c. limit followed by a <space> and a plus indicates that we can have the
1146             LIMIT clause followed by one or more spaces.
1147 
1148             d. [0-9]+ indicates that we can have one or more digits which specify the
1149             actual limit number for the LIMIT clause.
1150 
1151             e. *$ indicates we can have zero or more spaces before the end of the
1152             input string.
1153 
1154          2. (?i) +limit +[0-9]+ +offset +[0-9]+ *$ explanation
1155             a.The first portion of this regex is exactly the same as the previous one
1156 
1157             b. <space>+offset indicates that OFFSET keyword can appear after one or more
1158             spaces after the limit number has been specified.
1159 
1160             c. [0-9]+ indicates that we can have one or more digits that specify the
1161             offset value for the OFFSET clause.
1162 
1163             d. *$ indicates we can have zero or more spaces before the end of the
1164             input string.
1165 
1166          | - indicates that the separator between the two different expressions.
1167     */
1168     public static final String LIMIT_CLAUSE_PATTERN =
1169          "(?i) +limit +[0-9]+ *$|(?i) +limit +[0-9]+ +offset +[0-9]+ *$";
